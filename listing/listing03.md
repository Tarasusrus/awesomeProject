Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
<nil>
false

В Go интерфейс определяет набор методов, которые определенный тип должен имплементировать. 
Пустой интерфейс (interface{}) не имеет методов, поэтому каждый тип считается удовлетворяющим пустому интерфейсу.

Функция Foo возвращает ошибку, которая является интерфейсом. 
Внутри функции возвращается значение nil типа *os.PathError, 
который приводится к типу error (не пустому интерфейсу), поскольку *os.PathError удовлетстворяет интерфейсу ошибки.

интерфейс в Go содержит пару: значение и тип. Если значение и тип равны nil, интерфейс равен nil. 
В нашем случае значение равно nil, а тип равен *os.PathError, который является nil. 
Поэтому интерфейс не равен nil, даже если его значение равно nil.

от почему программа выводит <nil> при печати err (потому что значение nil),
 но false при сравнении с nil (потому что интерфейс, в целом, не равен nil).
```
